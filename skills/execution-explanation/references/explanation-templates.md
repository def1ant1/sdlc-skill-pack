# Explanation Templates Reference

## Audience Tier Templates

### Tier 1: Executive (C-Suite / Board)

**Characteristics:** Business outcomes focus, minimal technical detail, decision-oriented.

```
Template EX-T1-001: Workflow Outcome Summary

---
## What Happened

[1-2 sentences: what the workflow did in business terms]
Example: "The AI processed Q1 customer feedback and identified 3 product improvements
with estimated $2.4M combined revenue impact."

## Business Impact

| Metric | Before | After | Change |
|--------|--------|-------|--------|
| [Key metric 1] | X | Y | +Z% |
| [Key metric 2] | X | Y | -Z% |

## Decisions Required

[If any human decisions are needed, list here with recommended option]
→ [Decision]: Recommend [Option A] because [one-sentence rationale].

## Risks Flagged

[Only if material risks were identified; otherwise omit this section]
```

---

### Tier 2: Manager / Domain Expert

**Characteristics:** Process-level understanding, able to evaluate correctness, needs enough detail to supervise.

```
Template EX-T2-001: Process Execution Summary

---
## Objective
[What was the workflow trying to accomplish]

## Steps Completed
1. [Step name]: [What it did] → [Output/result]
2. [Step name]: [What it did] → [Output/result]
...

## Key Decisions Made
- [Decision point]: Selected [option] because [brief rationale]
  Alternatives considered: [Option B (rejected: reason), Option C (rejected: reason)]

## Outputs Produced
| Output | Location | Format | Notes |
|--------|----------|--------|-------|
| [name] | [path/URI] | [type] | [any issues] |

## Issues Encountered
- [Issue]: [How it was handled]

## Next Steps
[What, if anything, requires follow-up from the manager]
```

---

### Tier 3: Engineer / Analyst

**Characteristics:** Technical accuracy required; wants to understand mechanism, not just outcome.

```
Template EX-T3-001: Technical Execution Report

---
## Workflow Execution Summary
- Workflow ID: WF-{id}
- Duration: {duration_seconds}s (started {start_time}, ended {end_time})
- Steps executed: {n_steps} ({n_successful} successful, {n_failed} failed, {n_skipped} skipped)
- Model used: {model_id} (tier: {capability_tier})

## Step-by-Step Trace

### Step {n}: {step_name}
- **Input:** {input_summary}
- **Operation:** {operation_description}
- **Output:** {output_summary}
- **Duration:** {step_duration_ms}ms
- **Confidence:** {confidence_score} ({confidence_method})
- **Tokens used:** {tokens_in} in / {tokens_out} out

[Repeat for each step]

## Data Flow
```mermaid
graph LR
    A[Input: {input_name}] --> B[Step 1: {name}]
    B --> C[Step 2: {name}]
    C --> D[Output: {output_name}]
```

## Error Details
{If any step failed:}
- **Step:** {step_name}
- **Error type:** {error_type}
- **Error message:** {message}
- **Recovery action taken:** {action}
- **Impact:** {whether it affected output}
```

### Tier 4: Auditor / Compliance

**Characteristics:** Evidence-focused, needs provenance chain, immutable record.

```
Template EX-T4-001: Compliance Audit Record

---
## Audit Record Header
- Audit Record ID: AUD-{id}
- Workflow ID: WF-{id}
- Generated at: {timestamp} UTC
- Generated by: execution-explanation skill v{version}
- Immutable hash: SHA-256:{hash}

## Authorization Chain
| Level | Principal | Action Authorized | Timestamp |
|-------|-----------|------------------|-----------|
| System | {policy_name} | Workflow class approved | {timestamp} |
| User | {user_id} | Specific execution approved | {timestamp} |
| Override | N/A | No overrides applied | — |

## Data Access Log
| Dataset | PII Classification | Access Type | Justification |
|---------|-------------------|-------------|---------------|
| {name} | {level} | READ | {purpose} |

## Decision Audit Trail
| Decision ID | At Step | Options | Selected | Criteria | Confidence |
|-------------|---------|---------|----------|----------|------------|
| DEC-001 | Step 3 | [A, B, C] | A | {criteria summary} | 0.87 |

## Output Certification
All outputs have been scanned for:
- [x] PII (result: {clean/redacted_N_items})
- [x] Constitutional compliance (result: {compliant/violations_N})
- [x] Factual claims (result: {n_verified} verified, {n_unverified} unverified)

## Chain of Custody
{Cryptographic signature or hash chain linking inputs → operations → outputs}
```

## Drill-Down Link Format

When inline detail is available for a referenced item, use this link format:

```
[View full step trace](#step-{step_id})
[View data contract for {dataset}](#data-contract-{contract_id})
[View model routing decision](#routing-decision-{decision_id})
[View confidence breakdown](#confidence-{request_id})
```

---

## Explanation Rendering Guidelines

| Content Type | Rendering Recommendation |
|---|---|
| Decision tree | Mermaid flowchart |
| Metric comparisons | Markdown table |
| Time series | ASCII sparkline or linked dashboard |
| Step trace | Collapsible sections (if interactive) |
| Data flow | Mermaid graph LR |
| Confidence scores | Numeric (0.00–1.00) + label (HIGH/MEDIUM/LOW) |
| Durations | Human-readable (e.g., "2m 34s" not "154000ms") |

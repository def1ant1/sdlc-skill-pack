# Onboarding Checklist

## Pre-Start (1 Week Before Day 1)

**IT / Systems**
- [ ] Laptop ordered and configured
- [ ] Email account created
- [ ] GitHub/GitLab access provisioned (correct team)
- [ ] Slack workspace invitation sent
- [ ] 1Password or password manager invitation sent
- [ ] VPN access configured

**HR / Admin**
- [ ] Offer letter countersigned on file
- [ ] I-9 / right to work verification scheduled
- [ ] Benefits enrollment email sent
- [ ] Payroll information collected
- [ ] Employee handbook sent

**Manager Prep**
- [ ] 30/60/90 day plan drafted
- [ ] First week schedule blocked (no external meetings week 1)
- [ ] Buddy / onboarding partner assigned
- [ ] Team introduction email drafted

---

## Day 1

- [ ] Laptop set up and working
- [ ] All accounts accessible (email, Slack, GitHub, 1Password)
- [ ] HR paperwork completed (I-9, direct deposit, benefits)
- [ ] Meet manager (60-min welcome 1:1)
- [ ] Meet onboarding buddy (30-min intro)
- [ ] Company overview session (mission, product, team structure)
- [ ] Security training completed (mandatory; Day 1)
- [ ] Dev environment setup begun (follow CONTRIBUTING.md)

---

## Week 1

**Engineering New Hires**
- [ ] Dev environment fully functional (`make dev` running)
- [ ] First PR merged (even if small — a fix or doc update)
- [ ] Codebase walkthrough with buddy or tech lead
- [ ] Architecture overview session (1h)
- [ ] Pair programming session with a team member

**All New Hires**
- [ ] Met all immediate team members (1:1 schedule for weeks 1–4)
- [ ] Completed all mandatory compliance trainings
- [ ] Understood team's current sprint / project context
- [ ] Access to all required tools confirmed
- [ ] 30-day goals agreed with manager (written, measurable)

---

## 30-Day Milestone Check-In

Manager reviews with new hire:
- [ ] 30-day goals completed or on track
- [ ] Key relationships established (can name ≥ 5 key collaborators)
- [ ] Cultural fit discussion — any concerns on either side?
- [ ] Process clarity — understands how work flows through the team
- [ ] Open questions resolved

**Engineering**: First meaningful feature contribution underway or completed.

---

## 60-Day Milestone Check-In

- [ ] Independent contributor — can take and complete tasks without constant guidance
- [ ] Understands the product deeply enough to contribute to design discussions
- [ ] Participates in team rituals (code review, planning, retrospective)
- [ ] 60-day goals reviewed; 90-day goals set

---

## 90-Day Milestone Check-In

- [ ] Fully productive member of the team
- [ ] Has shipped at least one meaningful deliverable
- [ ] Understands how their role contributes to company goals (OKRs)
- [ ] Initial performance check-in complete (bi-directional feedback)
- [ ] Any issues or concerns from either side surfaced and addressed

---

## Onboarding Completion

Onboarding is considered complete at 90 days when:
- All 90-day milestones are met
- Security training fully completed and certified
- Employee is on standard performance review cadence
- Onboarding record closed in workforce-management

---

## Buddy Responsibilities

The onboarding buddy (a peer, not the manager) is responsible for:
- Daily check-in for the first 2 weeks
- Answering "how do we do X here?" questions without judgment
- Introducing the new hire to informal norms (how Slack works, how decisions get made)
- Flagging any concerns to HR or the manager confidentially
- Meeting with the new hire at 30 and 60 days to check in informally

# Explanation Formats Reference

## Overview

Audience tier definitions, explanation templates per tier, drill-down link format,
visualization embedding syntax, and output length guidelines for the explainability skill.

---

## Audience Tier Definitions

| Tier | Audience | Technical Depth | Max Length | Focus |
|---|---|---|---|---|
| Executive | Board, C-suite | None | 3 sentences | Outcome + business impact |
| Manager | Department heads, program managers | Low | 1–2 paragraphs | Decision + key rationale |
| Analyst | Data analysts, domain experts | Medium | Full page | Methodology + evidence |
| Engineer | Developers, platform team | Full | Unlimited | Step-by-step trace + inputs/outputs |

---

## Explanation Templates by Tier

### Executive Tier Template

```
[Workflow name] completed [successfully/with escalation] on [date].
The AI [action taken: reviewed / generated / analyzed / recommended] [subject],
resulting in [key outcome with business metric if available].
[If escalated: Human review was required because [one-sentence reason].]
```

**Example:**
> The contract review workflow completed successfully on 2026-05-07. The AI analyzed
> the vendor NDA and flagged 3 non-standard clauses for legal review, reducing manual
> review time by an estimated 60%. No escalation was required.

---

### Manager Tier Template

```
## What happened
[2-3 sentence summary of the workflow objective and outcome]

## Key decisions made
- [Decision 1]: [rationale in one sentence]
- [Decision 2]: [rationale in one sentence]

## Outcome
[Quantified result where available. Status: COMPLETED / ESCALATED / FAILED.]

[If escalated: ## Why human review was needed
[1 paragraph explanation of the specific decision that required human judgment]]
```

---

### Analyst Tier Template

```
## Objective
[Full statement of the workflow objective and constraints]

## Methodology
[Description of the approach: data sources used, reasoning method applied, models invoked]

## Evidence and data
[Key data points, retrieved documents, metric values that informed decisions]

## Decision logic
[Step-by-step explanation of how evidence led to each key decision]

## Outcome and confidence
[Final output with confidence level and known limitations]

## Drill-down references
[Links to Engineer-tier trace and visualization artifacts]
```

```
**Definition version:** X.Y.Z
**Total steps:** N | **Duration:** NNN ms | **Checkpoints:** N

### Step-by-step execution

#### Step 1: [Step name]
- **Skill invoked:** [skill-name]
- **Inputs:** [key input values]
- **Outputs:** [key output values or summary]

- **Status:** COMPLETED

[Repeat for each step]

### State at completion
[Final variable bindings relevant to the outcome]

### Deviations from expected
[Any retries, replanning events, or escalations with full detail]
```

## Drill-Down Link Format

Explanations at higher tiers link to more detailed tiers using structured references:

```markdown
[View detailed methodology →](explanation://WF-20260507-042/analyst)
[View full execution trace →](explanation://WF-20260507-042/engineer)
[View reasoning visualization →](visualization://WF-20260507-042/goal-tree)
```

The explanation service resolves these links to the appropriate stored explanation
document or generates one on demand if not yet cached.

---

## Visualization Embedding Syntax

Visualizations generated by reasoning-visualization are embedded using:

```markdown
![Goal tree for WF-20260507-042](viz://WF-20260507-042/goal-tree.mermaid)
```

Supported visualization types:

| Type | Format | Use Case |
|---|---|---|
| goal-tree | Mermaid flowchart | HTN decomposition of an objective |
| reasoning-chain | Mermaid flowchart | Step-by-step inference trace |
| causal-graph | DOT/Graphviz | Causal attribution network |
| execution-timeline | Gantt (Mermaid) | Workflow step timing |

## Decision Rationale Format

For any significant decision point, the rationale is structured as:

```yaml
decision_rationale:
  decision: "Select standard model tier for this task"
  evidence:
    - "Task complexity score: 4/10 (within Standard tier range)"
    - "Latency budget: 2,000ms (Standard P95: 1,200ms — within budget)"
    - "Cost ceiling: $0.01/request (Standard: $0.003 — within budget)"
  alternatives_considered:
    - option: "Advanced tier"
      rejected_because: "No evidence of complexity requiring Advanced; 3× cost premium unjustified"
  confidence: 0.92
```

## Output Length Guidelines

| Tier | Executive Summary | Body | Supporting Detail |
|---|---|---|---|
| Executive | 1–3 sentences | None | None |
| Manager | 2 sentences | 1–3 paragraphs | Optional bullet list |
| Analyst | 2 sentences | Full page | Referenced appendix |
| Engineer | 2 sentences | Unlimited | Full trace inline |

Explanations exceeding tier length limits are automatically summarized with a
"show more" reference to the full document.

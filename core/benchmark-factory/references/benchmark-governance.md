# Benchmark Governance Reference

## Overview

Defines the benchmark lifecycle stages, governance roles, versioning policy, quality gates
for certification, and benchmark registry format for the benchmark-factory.

---

## Benchmark Lifecycle Stages

```
[Draft] → [Under Review] → [Certified] → [Active] → [Deprecated] → [Retired]
```

### Stage Definitions

| Stage | Description | Who Can Advance |
|---|---|---|
| Draft | Initial benchmark created; not yet evaluated | Benchmark author |
| Under Review | Submitted for governance review; quality checks running | Governance committee |
| Certified | Passed all quality gates; approved for production use | Governance committee |
| Active | Being used in model evaluation; receiving usage metrics | N/A (automatic) |
| Deprecated | Superseded or coverage gaps identified; still usable | Governance committee |
| Retired | No longer used; weights removed from routing table | Governance committee |

---

## Governance Roles

| Role | Responsibilities |
|---|---|
| Benchmark Author | Creates benchmark, writes specification, submits for review |
| Capability Reviewer | Evaluates benchmark coverage of target capability |
| Safety Reviewer | Ensures no harmful or offensive content in benchmark items |
| Calibration Reviewer | Verifies difficulty distribution and contamination checks |
| Governance Committee | Final approval for Certified and Retired transitions |

---

## Quality Gates for Certification

All gates must pass before a benchmark transitions from Under Review to Certified:

### Gate 1: Coverage Adequacy

- Minimum 100 items total per benchmark
- Minimum 10 items per difficulty tier (Easy/Medium/Hard/Expert)
- Capability coverage ≥ 80% of the target capability's sub-skill taxonomy

### Gate 2: Correctness Verification

- ≥ 95% of items have unambiguous, verifiable reference answers
- Zero items with disputed or multi-answer reference answers (unless multi-answer format is declared)
- Correctness review completed by ≥ 2 independent reviewers

### Gate 3: Contamination Check

- < 5% overlap with any known public benchmark (similarity threshold: cosine > 0.90)
- < 1% overlap with training data of production models (where known)
- Contamination check tool must be run and report attached to certification record

### Gate 4: Difficulty Calibration

- Empirical difficulty distribution within ±15% of target (using reference model set)
- No difficulty tier with zero-shot pass rate > 95% (ceiling effect) or < 5% (floor effect)
- Calibration run must use ≥ 3 reference models spanning at least 2 capability tiers

### Gate 5: Safety Review

- Zero items that could be considered harmful, offensive, or privacy-violating
- Zero items that could be used to elicit harmful model outputs
- Safety reviewer sign-off required

## Versioning Policy

Benchmark versions follow: `MAJOR.MINOR.PATCH`

| Change Type | Version Bump | Backward Compat |
|---|---|---|
| New items added (no removals) | MINOR | Yes |
| Items removed or answers changed | MAJOR | No — requires re-evaluation |
| Metadata updates only (tags, descriptions) | PATCH | Yes |
| Difficulty re-labeling | MINOR | Yes (scores comparable) |
| Category taxonomy change | MAJOR | No — breaks trend analysis |

**Policy:** A MAJOR version bump invalidates historical comparison; all trend analysis
resets from the new major version's baseline.

## Benchmark Registry Format

```yaml
benchmark:
  benchmark_id: "BM-REASONING-001"
  name: "Multi-step Logical Reasoning"
  version: "2.1.0"
  capability: "logical-reasoning"
  stage: "Active"

  statistics:
    total_items: 500
    items_by_difficulty:
      Easy: 125
      Medium: 175
      Hard: 150
      Expert: 50
    adversarial_items: 50        # Items targeting known failure modes
    contamination_rate: 0.012    # 1.2% overlap with known public sets

  calibration:
    reference_models_used: ["Model-A", "Model-B", "Model-C"]
    empirical_difficulty_distribution:
      Easy: 0.23                 # Actual pass rate on Easy items
      Medium: 0.48
      Hard: 0.71
      Expert: 0.89

  governance:
    certified_by: "Governance Committee"
    certified_at: "2026-03-01"
    next_review_date: "2026-09-01"
    author: "platform-team"
    safety_reviewer: "safety-team"

  usage:
    times_used_in_evaluation: 47
    first_used: "2026-03-05"
    last_used: "2026-05-06"

  provenance:
    source_description: "Generated by benchmark-generation skill v1.2.0"
    contamination_check_tool_version: "1.0.3"
    calibration_run_id: "CAL-20260228-001"
```

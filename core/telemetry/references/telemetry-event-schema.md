# Telemetry Event Schema

## Base Event Structure

All telemetry events share this base structure:

```json
{
  "event_id": "uuid4",
  "event_type": "noun.verb",
  "service": "<emitting skill or service name>",
  "correlation_id": "<workflow or request id>",
  "timestamp": "ISO8601 UTC",
  "level": "info | warn | error | critical",
  "payload": {},
  "tags": {},
  "actor": {
    "type": "skill | agent | operator | system",
    "id": "<actor identifier>"
  }
}
```

**Naming convention**: `noun.verb` in snake_case. Past tense for completed actions.
Examples: `workflow.started`, `model.promoted`, `incident.declared`, `skill.activated`.

## Standard Event Types

### Workflow Events

```json
// workflow.started
{
  "event_type": "workflow.started",
  "payload": {
    "workflow_id": "WF-YYYYMMDD-NNN",
    "workflow_name": "<name>",
    "trigger": "<trigger type>",
    "input_hash": "<sha256 of inputs>"
  }
}

// workflow.step_completed
{
  "event_type": "workflow.step_completed",
  "payload": {
    "workflow_id": "WF-YYYYMMDD-NNN",
    "step_id": "<step>",
    "skill": "<skill name>",
    "duration_ms": N,
    "output_hash": "<sha256>"
  }
}

// workflow.completed | workflow.failed
{
  "event_type": "workflow.completed",
  "payload": {
    "workflow_id": "WF-YYYYMMDD-NNN",
    "duration_ms": N,
    "steps_completed": N,
    "steps_failed": N,
    "approval_events": N
  }
}
```

### AI Inference Events

```json
// model.inference_requested
{
  "event_type": "model.inference_requested",
  "payload": {
    "model": "claude-sonnet-4-6",
    "prompt_id": "<prompt registry id>",
    "prompt_version": "1.2.0",
    "input_tokens": N,
    "max_tokens": N,
    "temperature": 0.0,
    "backend": "anthropic_api | local_vllm | local_ollama"
  }
}

// model.inference_completed
{
  "event_type": "model.inference_completed",
  "payload": {
    "model": "<model>",
    "input_tokens": N,
    "output_tokens": N,
    "latency_ms": N,
    "cost_usd": X.XXXXXX,
    "cache_hit": true,
    "stop_reason": "end_turn | max_tokens | tool_use"
  }
}
```

### Security Events

```json
// security.action_blocked
{
  "event_type": "security.action_blocked",
  "level": "warn",
  "payload": {
    "requested_action": "<action>",
    "reason": "insufficient_authority | policy_violation | sandbox_limit",
    "required_level": "level-3",
    "actor_level": "level-1"
  }
}

// security.approval_granted | security.approval_denied
{
  "event_type": "security.approval_granted",
  "payload": {
    "workflow_id": "WF-YYYYMMDD-NNN",
    "action": "<action being approved>",
    "approver": "<operator id>",
    "level": "level-2"
  }
}
```

### Skill Events

```json
// skill.invoked
{
  "event_type": "skill.invoked",
  "payload": {
    "skill": "<skill name>",
    "version": "1.0.0",
    "workflow_id": "WF-YYYYMMDD-NNN",
    "input_size_bytes": N
  }
}

// skill.gap_detected
{
  "event_type": "skill.gap_detected",
  "level": "warn",
  "payload": {
    "gap_id": "GAP-YYYYMMDD-NNN",
    "gap_type": "<type>",
    "severity": "CRITICAL | HIGH | MEDIUM | LOW",
    "skill": "<skill name>"
  }
}
```

## Required Fields per Event

| Field | Requirement | Notes |
|---|---|---|
| `event_id` | Always required | UUID4; generated by emitter |
| `event_type` | Always required | noun.verb format |
| `service` | Always required | Emitting skill name |
| `correlation_id` | Required if in workflow | Links events within a workflow |
| `timestamp` | Always required | UTC ISO8601 |
| `level` | Always required | Default: info |

---

## PII Policy

Never include in telemetry events:
- Customer names (use anonymized account IDs instead)
- Email addresses
- API keys or secrets
- Salary or compensation data
- Health data
- Authentication tokens

Log anonymized identifiers only. Telemetry logs may be stored in external systems.
